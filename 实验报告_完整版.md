## 1. 实验简介

### 1.1 竞赛背景

本实验选择 Kaggle 平台上的 [Store Sales - Time Series Forecasting](https://www.kaggle.com/competitions/store-sales-time-series-forecasting) 竞赛。该竞赛要求参赛者使用 2013 年 1 月 1 日至 2017 年 8 月 15 日期间厄瓜多尔 54 家 Favorita 连锁超市门店的日常销售数据，预测 2017 年 8 月 16 日至 8 月 31 日（共 16 天）每家门店每种商品品类的销售额。

### 1.2 任务特点

- **多维度层次结构**：54 个门店（store）× 33 个商品品类（family）× 约 1684 天 = 超过 300 万条训练样本
- **复杂的外部因素**：原油价格波动、节假日与事件日历、2016 年厄瓜多尔大地震
- **极端的稀疏性**：大量 store-family 组合销量持续为零（已关店或不出售该品类）
- **时序依赖**：销售额存在明显的周周期、月周期和年周期模式
- **评估指标**：RMSLE（Root Mean Squared Logarithmic Error）

![销售额分布](experiment_report/v1/eda/01_sales_distribution.png)

### 1.3 解题思路概述

本实验设计了两套方案，呈迭代关系：

**V7（基线方案）**：传统机器学习路线。手工构建 146 维特征（包括 Periodogram 驱动的傅里叶特征、STL 风格时序分解、Rank 编码、油价/节假日/地震等外部特征），使用 LightGBM × 3 + XGBoost × 3 + CatBoost 七模型集成，配合 Optuna 超参搜索和 Family 自适应后处理融合。验证集 RMSLE 达到 0.3876。

**V8（最终方案）**：在 V7 的经验基础上转向 Darts 全局模型路线。核心思想是"每个商品品类训练一个模型，54 个门店序列共享模型参数"，利用全局模型自动学习时序模式，减少手工特征工程依赖。引入双窗口平均融合（全历史 + 2015 年后）和基于线性/log 空间的穷举权重搜索。最终 Leaderboard 得分 **0.03792**。

## 3. 程序框架

### 3.1 整体架构

本实验最终方案 V8 的程序框架如下：

```
数据加载 (locate_dataset → ingest_source_files)
    │
    ├─ 数据预处理 (assemble_modeling_table)
    │   ├─ 补齐缺失日期（MultiIndex reindex）
    │   ├─ 油价插值（线性插值）
    │   ├─ 交易数据插值 + 零销量店铺归零
    │   ├─ 节假日清洗 → one-hot 编码
    │   └─ 日历特征提取 + 字符串编码
    │
    ├─ Darts 序列构建
    │   ├─ extract_target_sequences() — 每 family 一组 TimeSeries
    │   │   └─ Pipeline: MissingValuesFiller → StaticCovariatesTransformer(OneHot) → log1p → Scaler
    │   └─ extract_covariate_sequences()
    │       ├─ Past covariates: transactions（带 7/28 天 MA）
    │       └─ Future covariates: oil, onpromotion, 日历特征, 节假日（带 7/28 天 MA）
    │
    ├─ GlobalForecastEngine（核心引擎）
    │   ├─ 模型配置: LightGBM × 4 lag + XGBoost × 4 lag
    │   │   └─ Lags: [63, 7, 365, 730] — 季度 + 周 + 年 + 两年
    │   ├─ 双窗口训练:
    │   │   ├─ full-history（2013-2017）
    │   │   └─ 2015+（仅 2015 年后，排除早期不稳定数据）
    │   └─ 预测融合: avg(avg(LGBM_full, LGBM_2015), avg(XGB_full, XGB_2015))
    │
    ├─ 权重搜索 (run_weight_search) — 可选
    │   ├─ 在 16 天回测验证集上穷举 4 组件权重
    │   ├─ Linear space: w1*p1 + w2*p2 + w3*p3 + w4*p4
    │   └─ Log space: expm1(w1*log1p(p1) + ...)
    │
    └─ 输出
        ├─ submission_v8_clean.csv（均匀权重）
        ├─ submission_v8_local_opt.csv（搜索最优权重）
        ├─ v8_local_blend_weights.csv（权重表）
        └─ experiment_report.zip（全部可视化图表）
```

### 3.2 核心设计思想

V8 相比 V7 的根本变化在于**从"手工特征 + 树模型"转向"全局时序模型"**：

| 维度 | V7（传统路线） | V8（全局模型路线） |
|------|---------------|-------------------|
| 特征来源 | 手工构建 146 维特征 | Darts 自动处理时序依赖 |
| 模型结构 | 7 个独立模型集成 | 每 family 一个全局模型，含 54 个子序列 |
| 时序建模 | 滞后特征 + 滚动统计 | Lags + 协变量自动关联 |
| 训练窗口 | 单一窗口（2014+） | 双窗口（全历史 / 2015+） |
| 融合策略 | 网格搜索三模型权重 | 网格搜索四组件权重（线性 + log 空间） |
| 验证窗口 | 31 天 | 16 天 |

## 4. 关键代码实现

### 4.1 V7 核心创新：Periodogram 驱动的傅里叶特征

V7 方案的核心独创性在于使用**周期图（Periodogram）分析**而非经验直觉来确定傅里叶特征的最佳频率。传统做法是"拍脑袋"选择 sin/cos 周期（如月、季、年），V7 对聚合后的每日总销量做 FFT 分析，在频域中寻找最强信号：

![Periodogram 频谱分析](experiment_report/v1/eda/08_periodogram.png)

从频谱图中可以清晰看到四个峰值对应的周期：**3.5 天（半周）、7 天（周）、30 天（月）、365 天（年）**。V7 据此精确设计多阶傅里叶特征：

```python
fourier_config = [
    (3.5,  3),   # 半周周期, 3阶 — 捕捉高频波动
    (7.0,  3),   # 周周期,   3阶 — 周末效应
    (30.0, 3),   # 月周期,   3阶 — 月度发薪周期
    (365.0, 2),  # 年周期,   2阶 — 年度季节性
]
for freq, order in fourier_config:
    for k in range(1, order + 1):
        data[f"fourier_sin_{freq}_{k}"] = np.sin(2 * k * np.pi * t / freq)
        data[f"fourier_cos_{freq}_{k}"] = np.cos(2 * k * np.pi * t / freq)
```

共生成 22 个傅里叶特征（= 2 × (3+3+3+2)），多阶设计允许模型捕获比单阶正弦波更复杂的非对称周期模式。

### 4.2 V7 核心创新：STL 风格时序分解

V7 使用指数加权移动平均（EWM，span=182）近似 LOESS 趋势，将销售序列分解为趋势、季节和残差三个分量：

```python
# Trend: 长周期 EWM
data["stl_trend"] = groupby_shifted_ewm("sales_log", span=182)
# Seasonal: 滚动均值减去趋势
data["stl_seasonal_7"]  = data["rm_7"]  - data["stl_trend"]
data["stl_seasonal_28"] = data["rm_28"] - data["stl_trend"]
# Residual: 原始值减去趋势和季节
data["stl_resid"] = sales_log - trend - seasonal_7
# Strength: 季节信号 vs 噪声的相对强度
data["stl_strength"] = |seasonal| / (|seasonal| + |residual|)
```

![STL 分解示例](experiment_report/v1/features/01_stl_decomposition.png)

分解后模型可以分别学习趋势、周期和残差三个维度的模式。特别是在地震（2016-04）前后，趋势分量的变化清晰可见。

### 4.3 V7 其他关键特征

**Rank 编码**：用平均销量排名替代原始的类别标签，避免高基数类别带来的稀疏性问题：

```python
data["family_rank"] = data["family"].map(family_mean_sales_rank)
data["store_rank"]  = data["store_nbr"].map(store_mean_sales_rank)
data["cluster_rank"] = data["cluster"].map(cluster_mean_sales_rank)
```

![Rank 分布](experiment_report/v1/features/02_rank_distributions.png)

**异常值 Clip**：按 store-family 维度截断 99.5% 分位数以上的极端值，防止离群点污染模型训练：

![异常值分析](experiment_report/v1/features/04_outlier_analysis.png)

**发薪日特征**：厄瓜多尔发薪日为每月 15 号和月末，V7 据此构建距离发薪日的天数和发薪日标记：

```python
data["days_to_payday"] = min(|day - 15|, |day - days_in_month|)
data["is_payday"] = (day == 15) | (day == days_in_month)
```

**关店检测 + 节假日 × 促销交互**：通过训练集末尾 28 天的销量总和检测疑似关店的 store 组合，构建促销与节假日的交互特征。

### 4.4 V8 核心创新：全局模型引擎

V8 最关键的代码模块是 `GlobalForecastEngine` 类（原约 300 行），其核心方法 `_produce_forecasts` 实现了完整的"训练 → 预测 → 逆变换 → 零销量截断"pipeline：

```python
def _produce_forecasts(self, models, train_seqs, pipe, p_covs, f_covs, truncate_before):
    # 1. 可选截断：drop_before 剔除早期数据
    if truncate_before is not None:
        boundary = pd.Timestamp(truncate_before) - pd.Timedelta(days=1)
        train_seqs = [s.drop_before(boundary) for s in train_seqs]

    # 2. 每个模型 fit + predict + inverse_transform
    for idx, mdl in enumerate(models):
        mdl.fit(series=train_seqs, past_covariates=p_covs, future_covariates=f_covs)
        raw_pred = mdl.predict(n=self.horizon, ...)
        restored = pipe.inverse_transform(raw_pred)  # Scaler⁻¹ → expm1

        # 3. 零销量店铺强制归零（稳健性处理）
        for j in range(n_stores):
            if train_seqs[j][-self.zero_window:].values().sum() == 0:
                restored[j] = zero_template

        # 4. Clip 负值 + 均匀集成
        cleaned = [p.map(self._clamp) for p in restored]
        for j in range(n_stores):
            running_ensemble[j] += cleaned[j] / len(models)

    return per_model_preds, running_ensemble
```

四个滞后阶数 `[63, 7, 365, 730]` 分别对应季度、周、年和两年的历史窗口，让全局模型能同时捕捉短期波动和长期趋势。

### 4.5 V8 核心创新：双空间权重搜索

V8 的权重搜索不同于 V7 仅在原始空间搜索，而是分别在**线性空间**和**对数空间**并行搜索，选取更优者：

```python
def optimize_component_weights(actual, parts, step=0.05, top_k=30):
    mat = np.vstack([parts[n] for n in names])       # [4, N] 预测矩阵
    log_mat = np.log1p(mat)                           # log 空间

    for weights in _walk_simplex(4, step):            # 穷举所有权重组合
        # 线性空间
        pred_linear = np.dot(weights, mat)            # 直接加权和
        # 对数空间
        pred_log = np.expm1(np.dot(weights, log_mat)) # log 空间加权后 expm1

    return results.sort_values("rmsle").head(top_k)
```

`_walk_simplex` 生成满足 ∑wᵢ=1 的所有权重组合，步长为 0.05 时约有 ~1771 种组合。

![权重搜索热力图](experiment_report/v2/models/01_blend_search.png)

Linear 空间和 Log 空间的最优权重通常在相近但略有不同的区域，Log 空间在销量跨越多个数量级时往往更优。

## 5. 最终性能评价

### 5.1 评价指标：RMSLE

本竞赛使用 **RMSLE（Root Mean Squared Logarithmic Error）** 作为评价指标：

$$\text{RMSLE} = \sqrt{\frac{1}{N} \sum_{i=1}^{N} \left( \ln(1 + \hat{y}_i) - \ln(1 + y_i) \right)^2}$$

RMSLE 先取对数再求均方根，因此对大值的惩罚比 MSE 轻，对小值的敏感度更高。这符合零售预测的业务场景：预测 100 件只卖出 10 件的错误比预测 1100 件卖出 1010 件的错误更严重。RMSLE 为 0 表示完美预测。

残差近似正态分布，均值接近 0，无显著偏斜。但有轻微的长尾现象（少量极端高销量的预测偏低）。

## 7. 不足

### 7.1 V7 方案的不足

1. **CatBoost 未发挥应有作用**：在 V7 特征集上 CatBoost 的 RMSLE 明显差于 LGB/XGB，导致三路融合退化为单模型。可能原因：CatBoost 对高维稠密特征（如 146 维）的优势不如 LGB/XGB 明显，且未针对 CatBoost 做独立超参搜索
2. **验证窗口不一致**：V7 使用 31 天窗口但竞赛只预测 16 天，验证分布与测试分布不完全匹配
3. **特征维度偏高**：146 维特征虽丰富但部分高度相关（如多阶滚动特征），可能引入冗余

### 7.2 V8 方案的不足

1. **单轮验证**：V8 的 `folds=1`，仅使用最后 16 天做回测，未做多折交叉验证来评估模型稳定性
2. **XGBoost 超参未搜索**：V8 的 XGBoost 超参通过环境变量设置，未像 V7 那样用 Optuna 自动搜索
3. **缺少 CatBoost 组件**：V7 的经验表明 CatBoost 在特定场景可能有互补作用，V8 未纳入
4. **全局模型的黑盒性**：Darts 全局模型内部机制（Shapley 值、注意力权重等）不如手工特征可解释

### 7.3 可改进方向

1. 引入**时间序列交叉验证**（TimeSeriesSplit），评估模型在多个时间切片上的稳定性
2. 对 V8 的 XGBoost 和 LightGBM 超参做**联合 Optuna 搜索**而非手动设置
3. 尝试将 V7 的**傅里叶特征**作为额外 future covariates 注入 V8 的 Darts 模型
4. 考虑**神经网络方案**（N-BEATS、TFT）作为第三组件加入融合

## 8. 心得体会

在本次实验中，我完整经历了从一个 Kaggle 竞赛的"查分→复现→改进→创新"的全过程，深刻体会到了以下几点：

1. **"先跑通再优化"的重要性**：V7 虽然最终不如 V8，但它快速建立了一个可运行的 baseline，让我在早期就能看到结果、发现问题。很多优化方向（如关店检测、异常值 clip、偏置修正）都是从 V7 的残差分析中发现的。

2. **全局模型 vs 手工特征的取舍**：V7 的 146 维手工特征非常"趁手"——你可以看懂每一个特征的含义，知道它在建模什么。V8 的 Darts 全局模型则更像一个黑盒——它自动学到了很多东西，但调试起来更困难。在实际应用中，两种路线的结合（如把手工特征作为 Darts 协变量）可能是最优解。

3. **工程细节决定上限**：本实验中有大量代码是"非核心但必须"的：GPU 自动检测、环境路径搜索、安全绘图包装器、报告自动打包。这些工程细节不会直接提升模型分数，但保证了代码在不同平台（Kaggle/AutoDL/本地）都能稳定运行，也大大降低了实验迭代的成本。

4. **竞赛驱动的学习**：通过参加实际竞赛，我对时序预测的认知从课本上的 ARIMA/指数平滑跳到了工业界真正在用的 LightGBM/XGBoost/Darts 全局模型。这种"做中学"的效率远高于纯理论学习。
